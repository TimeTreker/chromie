# Chromie Engineering Context

Concise handoff for coding agents. Current status authority:
[docs/STATUS.md](docs/STATUS.md). Current milestone: **M13**.

## Runtime boundary

```text
Host Orchestrator
  microphone, VAD, session/conversation state, interruption, playback,
  trusted Skill Runtime, provider cancellation, correlated JSONL evidence

Docker
  chromie-asr     speech recognition
  chromie-router  deterministic and optional LLM routing
  chromie-agent   conversation, native interaction, capability registry,
                  TaskGraph planning and execution APIs
  chromie-llm     Ollama
  chromie-tts     streaming speech synthesis

Separate deployment
  Soridormi MCP   named robot skills, embodied safety, simulator/hardware
```

The legacy host hardware daemon is mock compatibility only. It currently
constructs `MockRobotDriver` and is not the M13 robot boundary.

## Current control flow

```text
microphone -> host VAD -> ASR -> Router
  -> Agent /run                                  compatibility mode
  -> Agent /interaction -> trusted Skill Runtime structured mode
       -> speech provider -> TTS -> playback
       -> Soridormi provider -> MCP -> simulator/robot
```

`POST /interaction` is strict and operational. It now uses native
`InteractionResponse` accumulation by default, validates the serialized contract,
and fails closed on malformed output. The legacy adapter remains available only
through explicit compatibility or fallback settings.

## Invariants

- Keep realtime audio and Skill Runtime coordination on the host.
- Keep embodied execution and physical safety in Soridormi.
- Keep stop, cancel, emergency, ignore/silence, and unusable-audio behavior
  deterministic.
- Reject raw motor, joint, torque, and actuator fields from model-facing
  contracts.
- Keep all risky execution gates default-off.
- Do not auto-dispatch LLM-planned TaskGraphs.
- Preserve request/graph-bound confirmation and monitor proofs.
- Keep physical work sequential even when non-physical parallelism is enabled.
- Warm the selected Ollama model before opening the microphone.
- Keep host-to-Agent timeout longer than Agent-to-Ollama timeout.
- Keep TTS model generation serialized unless its mutable backend is redesigned.
- Launch with `python -m orchestrator.orchestrator` from repository root.

## Configuration

```text
.env.common + env/profiles/<profile>.env + .env.local -> .env.runtime
```

Host `orchestrator/.env.local` fills values not already present in the process.
Container-to-container URLs use Compose service names; host URLs use published
loopback ports.

## Verification

```bash
./scripts/run_tests.sh
python scripts/check_docs.py
```

Use `docs/ACCEPTANCE.md` before making target or hardware claims. The guided
runner is `scripts/m13_voice_acceptance.py`; its evidence is checked by
`scripts/verify_m13_evidence.py`. Candidate packaging is generated by
`scripts/prepare_alpha_release.py` and must remain non-publishable while
`release/compatibility.json` lists closure blockers.
